#### <u><b> Version Control System - GIT (1) </b></u>


### 버전 컨트롤

CVS, SVN, GIT 이렇게 세개가 있는데, GIT이 지금이다.

### 1. 깃 설치 확인

~~~
bash-4.4$ git
usage: git [--version] [--help] [-C <path>] [-c <name>=<value>]
           [--exec-path[=<path>]] [--html-path] [--man-path] [--info-path]
           [-p | --paginate | -P | --no-pager] [--no-replace-objects] [--bare]
           [--git-dir=<path>] [--work-tree=<path>] [--namespace=<name>]
           <command> [<args>]

These are common Git commands used in various situations:

start a working area (see also: git help tutorial)
   clone      Clone a repository into a new directory
   init       Create an empty Git repository or reinitialize an existing one

work on the current change (see also: git help everyday)
   add        Add file contents to the index
   mv         Move or rename a file, a directory, or a symlink
   reset      Reset current HEAD to the specified state
   rm         Remove files from the working tree and from the index

examine the history and state (see also: git help revisions)
   bisect     Use binary search to find the commit that introduced a bug
   grep       Print lines matching a pattern
   log        Show commit logs
   show       Show various types of objects
   status     Show the working tree status

grow, mark and tweak your common history
   branch     List, create, or delete branches
   checkout   Switch branches or restore working tree files
   commit     Record changes to the repository
   diff       Show changes between commits, commit and working tree, etc
   merge      Join two or more development histories together
   rebase     Reapply commits on top of another base tip
   tag        Create, list, delete or verify a tag object signed with GPG

collaborate (see also: git help workflows)
   fetch      Download objects and refs from another repository
   pull       Fetch from and integrate with another repository or a local branch
   push       Update remote refs along with associated objects

'git help -a' and 'git help -g' list available subcommands and some
concept guides. See 'git help <command>' or 'git help <concept>'
to read about a specific subcommand or concept.
~~~

### 2. 버전컨트롤을 위한 디렉토리 생성하기
~~~
bash-4.4$ mkdir gitdir
bash-4.4$ ls
gitdir
bash-4.4$ cd gitdir/
~~~

### 2. 현재 디렉토리를 버전컨트롤하겠다고 선언하기
~~~
bash-4.4$ git init
Initialized empty Git repository in /home/work/gitdir/.git/
~~~

### 2.1 .git이라는 디렉토리는 버전컨트롤을 하며 생성되는 여러 임시 파일들이 저장되는 곳. 해당 디렉토리가 버전컨트롤 됨을 의미함.
~~~
bash-4.4$ ls -al
total 12
drwxr-xr-x    3 work     work          4096 Apr 18 06:10 .
drwxr-xr-x    3 work     work          4096 Apr 18 06:08 ..
drwxr-xr-x    7 work     work          4096 Apr 18 06:10 .git
~~~

### 3. vim에디터를 사용해서 f1.txt라는 파일 만들기.(one_one이라고 입력하기)

~~~
bash-4.4$ vim f1.txt
bash-4.4$ ls -al
total 16
drwxr-xr-x    3 work     work          4096 Apr 18 06:13 .
drwxr-xr-x    3 work     work          4096 Apr 18 06:08 ..
drwxr-xr-x    7 work     work          4096 Apr 18 06:10 .git
-rw-r--r--    1 work     work            33 Apr 18 06:13 f1.txt
~~~


### 3.1. f1.txt 파일 내용 확인하기
~~~
bash-4.4$ cat f1.txt
one_one
~~~

### git 상태 확인하기

~~~
bash-4.4$ git status
On branch master

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)

        f1.txt

nothing added to commit but untracked files present (use "git add" to track)
~~~

### 4. git stage area에 파일 등록하기
~~~
bash-4.4$ git add f1.txt
~~~

### 4.1. git 상태 확인하기

~~~
bash-4.4$ git status
On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)

        new file:   f1.txt
~~~

### 5. git 사용자 설정하기(한번만하면됨)
~~~
bash-4.4$ git config --global user.name davidlee
bash-4.4$ git config --global user.email dl0717@gmail.com
~~~

### 6. git log 확인하기

~~~
bash-4.4$ git log
fatal: your current branch 'master' does not have any commits yet
~~~

### 7. commit 하기(맨 윗줄에 'logone_logone' 이라고 적기)
~~~
bash-4.4$ git commit f1.txt
[master (root-commit) 9de3cd4] logone_logone
 1 file changed, 1 insertion(+)
 create mode 100644 f1.txt
~~~

### 8. git 상태 확인하기
~~~
bash-4.4$ git status
On branch master
nothing to commit, working tree clean
~~~


### 8. git log 확인하기(작성자 포함해서 commit 한 내용을 확인할 수 있음)
~~~
bash-4.4$ git log
~~~

### 9. vim에디터를 사용해서 f1.txt 파일 수정하기.(one_two라고 수정하기)
~~~
bash-4.4$ vim f1.txt
bash-4.4$ cat f1.txt
two_two
~~~

### 10. git 상태 확인하기
~~~
bash-4.4$ git status
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

        modified:   f1.txt

no changes added to commit (use "git add" and/or "git commit -a")
~~~

### 11. 수정된 f1.txt파일을 git stage area에 파일 등록하기
~~~
bash-4.4$ git add f1.txt
bash-4.4$ git status
On branch master
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

        modified:   f1.txt
~~~

### 12. 수정된 f1.txt파일을 commit 하기(맨 윗줄에 logone_logtwo라고 적기)
~~~
bash-4.4$ git commit f1.txt
[master b0308b3] logtwo_logtwo
 1 file changed, 1 insertion(+), 1 deletion(-)
~~~

### 10. git 상태 확인하기
~~~
bash-4.4$ git status
On branch master
nothing to commit, working tree clean
~~~
